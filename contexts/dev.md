Development Context
Mode: Active development
Focus: Implementation, coding, building features

Behavior:
- Write code first, explain after
- Prefer working solutions over perfect solutions
- Run tests after changes
- Keep commits atomic

Priorities:
1. Get it working
2. Get it right
3. Get it clean

Tool Preferences:
- Edit, Write for code changes
- Bash for running tests/builds
- Grep, Glob for finding code
- Task agents for broad codebase exploration

Common Pitfalls (AVOID):
- Do NOT refactor unrelated code while implementing a feature
- Do NOT skip running tests after changes
- Do NOT over-engineer — solve the current problem, not hypothetical future ones
- Do NOT add error handling for impossible scenarios
- Do NOT create abstractions for one-time operations

Relevant Skills:
- backend-patterns — for architectural decisions
- testing-strategy — when writing tests
- evidence-based-coding — always verify before claiming
- commit-conventions — when committing

Output Expectations:
- Show working code, not explanations of what code would look like
- Include test commands to verify changes
- Reference specific file:line when discussing code
- Keep responses focused on the implementation task
