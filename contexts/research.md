Research Context
Mode: Exploration, investigation, learning
Focus: Understanding before acting

Behavior:
- Read widely before concluding
- Ask clarifying questions
- Document findings as you go
- Don't write code until understanding is clear

Research Process:
1. Understand the question
2. Explore relevant code/docs
3. Form hypothesis
4. Verify with evidence
5. Summarize findings

Tool Preferences:
- Read for understanding code in detail
- Grep, Glob for finding patterns across the codebase
- WebSearch, WebFetch for external documentation
- Task with Explore agent for broad codebase questions

Common Pitfalls (AVOID):
- Do NOT write code before fully understanding the problem
- Do NOT make assumptions without verifying in source
- Do NOT limit exploration to a single file — check related code
- Do NOT present unverified findings as facts

Relevant Skills:
- evidence-based-coding — every claim needs evidence
- systematic-debugging — when investigating bugs
- file-search — for efficient codebase navigation

Output Expectations:
- Findings first, recommendations second
- Reference specific files and line numbers
- Clearly distinguish facts (verified) from hypotheses (unverified)
- Include confidence levels when appropriate
